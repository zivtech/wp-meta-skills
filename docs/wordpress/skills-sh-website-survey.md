## Screening Notes

- The official `wordpress/agent-skills` rows remain the primary upstream candidate family because they cover plugins, blocks, themes, REST, Interactivity API, Abilities API, performance, WP-CLI/ops, Playground, PHPStan, plugin-directory guidelines, and blueprints.
- `jeffallan/claude-skills`, `jezweb/claude-skills`, `bobmatnyc/claude-mpm-skills`, `sickn33/antigravity-awesome-skills`, `bartekmis/wordpress-performance-best-practises`, and `elvismdev/claude-wordpress-skills` are useful community comparators for breadth, page builders, security, block themes, plugin fundamentals, and performance.
- `automattic/wordpress-activitypub`, `bonny/wordpress-simple-history`, and `emdash-cms/emdash` appear to be plugin/project-specific skill sets. They are useful as narrow release/compliance comparators but should not define the general WordPress V1 lifecycle suite.
- Reuse remains blocked for any candidate without a verified compatible license, even if the skill ranks highly on skills.sh.
